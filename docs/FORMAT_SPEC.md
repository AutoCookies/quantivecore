# QuantCore Packed Tensor Format Specification

## Version

- Format version: `v1`
- API version header: `include/quantcore/version.hpp`

## Common Rules (v1)

- Logical tensor layout is row-major.
- Data are packed in `uint64_t` blocks.
- Each block stores 64 logical elements.
- Bit indexing inside each block is LSB-first:
  - logical element `k` maps to bit `(k % 64)` in block `(k / 64)`.
- For row `r`, element `c` is read from block index:
  - `r * blocks_per_row + (c / 64)`

## Binary Format (v1)

- Domain: `{ -1, +1 }`
- Encoding:
  - bit `0` => `-1`
  - bit `1` => `+1`
- Packing granularity: 1 bit/element.
- Tail handling when `K` is not divisible by 64:
  - unused bits in final block are ignored and masked out in kernels.

### Binary Dot Product Identity

Given packed vectors `A`, `B` with K elements:

`dot(A,B) = 2 * popcount(~(A XOR B)) - K`

Applied block-wise with a valid-bit mask on the tail block.

## Ternary Format (v1)

- Domain: `{ -1, 0, +1 }`
- Two bitplanes per tensor row:
  - `positive_mask`
  - `negative_mask`
- Encoding per element:
  - `+1` => `positive=1, negative=0`
  - `-1` => `positive=0, negative=1`
  - `0`  => `positive=0, negative=0`
- Invalid state (`positive=1, negative=1`) is not generated by reference packers.

### Ternary Dot Product Identity

`dot(A,B) = popcnt(A_pos & B_pos) + popcnt(A_neg & B_neg) - popcnt(A_pos & B_neg) - popcnt(A_neg & B_pos)`

Applied block-wise with valid-tail masking.
